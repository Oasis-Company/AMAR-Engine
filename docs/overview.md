# AMAR Engine 系统概述

## 1. 什么是 AMAR Engine？

AMAR Engine (简称 AME) 是一个**现实映射与物理重构系统**，而非传统的 3D 渲染引擎。它通过对高维空间点云的重采样，结合多模态语义注入，在数字维度内"强行"实例化具有物理实权的实体。

### 核心使命

**PHYSICAL INSTANTIATION OF REALITY**（现实的物理实例化）

### 系统愿景

在 AME 的世界观里，万物皆可被**实例化 (Instantiation)**，而物理规律由底层引擎 **AMAR GENIS** 绝对统治。AME 系统旨在创建一个能够真实反映和模拟现实世界的数字生态系统，其中的每个实体都具有与现实世界高度一致的物理属性和行为。

## 2. 系统组成

AME 系统由以下核心组件组成：

### 2.1 AME Scanner

- **职能**：空间骨架捕获
- **核心技术**：密度塌陷处理、OBB 拟合
- **输出**：提供实体的几何约束
- **位置**：3DGS 路径的执行者

### 2.2 AMAR ENGINE

- **职能**：系统核心，包含归一中心 (Admin)
- **核心技术**：元类系统、AEID 系统、Skills 系统
- **输出**：将三条路径的数据坍缩为唯一的物理实体
- **位置**：系统的中枢，数字黑洞

### 2.3 AMAR GENIS

- **职能**：物理引擎，物理规律的执行者
- **核心技术**：多种物理求解器、渲染系统、传感器系统
- **输出**：实现实体的物理实例化和行为模拟
- **位置**：底层物理引擎，AME 世界的物理法官

### 2.4 元类系统

- **职能**：定义实体的物理属性和行为规则
- **核心技术**：根元类、标准元类、元类继承
- **输出**：为实体提供自洽的行为规则
- **位置**：系统的遗传基因

## 3. 技术创新

### 3.1 三路递归架构

AME 系统采用三路递归架构，以 Admin 为中枢，将三条异构路径的数据坍缩为唯一的物理实体：

1. **3DGS 路径**：通过密度加权采样算法导航 3D 辐射场，提取空间真相
2. **媒体输入路径**：处理跨频谱图像、时序视频、声纹数据，确定物体的表面材质流与声学阻抗
3. **LLM API 路径**：将自然语言碎片转化为元类描述，注入物理常量

### 3.2 元类系统

元类是 AME 系统的遗传基因，其演化逻辑如下：

- **演化趋势**：具体 → 抽象 → 自洽
- **根元类**：物理世界的公理（如：空间、时间步、光滑度）
- **标准元类**：具备交互潜力的实体模板（如：容器、轮子、液体）
- **表现形式**：实体在不同坐标系下的存在状态（如：笛卡尔 Room、相对感官 Room）

### 3.3 物理句柄绑定

实现 3DGS 视觉层与物理碰撞层的非同步耦合，支持拓扑突变，确保视觉表现与物理行为的一致性。

### 3.4 实时元类注入

AMAR GENIS 直接解析 Admin 下发的元类标签，无需重启模拟即可实时改变物体的物理常数。
